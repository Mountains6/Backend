# NestJS
NestJS - фреймворк для создания масштабируемых серверных приложений. Он написан на TypeScript и
из коробки предлагает строгую и предсказуемую архитектуру
NestJS использует Express внутри себя по умолчанию в качестве базового HTTP сервера, а Nest выступает
в роли мощной архитектурной надстройки
Основные элементы:

1. Модули(@Module) - главные кирпичики приложения. Позволяют изолировать логику и инкапсулирует
 внутри себя контроллеры и провайдеры(сервисы)
2. Контроллеры(@Controller) - отвечают за обработку запросов и возвращение ответов клиенту 
   (@Get("/users"), @Post())
3. Провайдеры/Сервисы(@Injectable) - содержат чистую бизнес-логику и внедряются в контроллеры

## NestJS и Express
Express: 
 - чистый JS (можно подключить TS)
 - каждый разработчик строит архитектуру сам
 - масштабируемость сложная и есть риск превращения кода в "спагетти"
 - нужно подключать много дополнительных инструментов, например для валидации

NestJS:
 - cразу на TS
 - модульная архитектура
 - высокая масштабируемость
 - большое количество встроенных инструментов
 
 ### Декораторы (@Module, @Controller, @Injectable, @Get )
 Декораторы - позволяют фреймворку автоматически настраивать маршруты, валидацию
 и зависимости
